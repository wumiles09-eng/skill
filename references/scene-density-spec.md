# Scene Density Spec — 量化爽点密度规格（真实样本反推）

> **本文是 `viral-formula.md` 的量化补集。** viral-formula 给出 hook/cliffhanger/karma 的*类型*与*定性*密度（"3-4 conflicts/episode"、"2 reversals/episode"）；本文给出基于 **25 部爆款真实视频反推**的*量化基线*、*流派分层*、*单集时序硬钉*、*付费卡点密度公式*与*可执行自检表*。
>
> **数据来源**：25 部 netshort.com / shorttv.live 爆款，下载真实视频 → ffmpeg 抽音 → faster-whisper ASR 转写 → 逐句时间戳。原始数据见 `references/research-evidence.md`。
>
> **核心结论**：爽点密度**强流派分化**（最高与最低差 3-4 倍），一刀切的"3-4 conflicts/集"会同时让复仇剧太疏、玄幻剧太密。必须按流派 + 集位（开头/中段/付费墙）给不同基线。
>
> **v3.0 新增**：①集时长修正为 90–210s（§1）；②整弧付费转化结构见姊妹篇 **`netshort-arc-mechanics.md`**（梯次身份揭晓 / netshort 付费墙=痛点峰值非真相曝光 / 反派升级阶梯 / 阶级羞辱词库 / 具象信物）。本文管"每集多密"，那篇管"整弧怎么排"。

## §1. 密度三档基线（按流派 + 首对白延迟）

真实样本的"段/分"（segs/min = ASR 句切分频率 ≈ 情节转折/对白交锋密度）和"首对白延迟"（开场多久才出第一句人声）呈现清晰的三档：

### 档位 A — 对白密集型（Dialogue-Dense）

| 维度 | 基线 |
|------|------|
| segs/min | **8 – 21** |
| words/min | 78 – 111 |
| 首对白延迟 | **0 – 10s** |
| 信息载体 | 高频短对白交锋（bullet dialogue） |
| 爽点驱动 | 打脸 / 反转 / 身份揭晓，靠"说"推进 |

**真实样本**：crown（宫斗，15.5–21.5 segs/min）、married-a-ceo（复仇契约婚，13.0）、mommy（情感悬疑，13.7）、bankrupt（复仇打脸，8.3）。

**写作含义**：开场 10s 内必须进入对白冲突，不能有纯画面铺垫；每 8–12s 一个对白回合或信息转折。

### 档位 B — 标准言情型（Standard Romance）

| 维度 | 基线 |
|------|------|
| segs/min | **5 – 7** |
| words/min | 51 – 69 |
| 首对白延迟 | 0 – 10s |
| 信息载体 | 对白 + 动作交替，留呼吸感 |
| 爽点驱动 | 萌宝上帝视角反差 / 契约婚 / 护妻护崽 |

**真实样本**：custody（抚养权+霸总，6.4）、adopted-baby（萌宝财神，6.1）、beautiful-ceo（霸总+萌宝，5.9）、ugly-girl-cooking（逆袭厨神，5.6）。

**写作含义**：每 10–12s 一个节拍，对白与动作戏/情绪特写交替；萌宝类靠"观众知道真身、配角不知"的反差持续蓄力。

### 档位 C — 奇观慢热型（Spectacle Slow-Burn）

| 维度 | 基线 |
|------|------|
| segs/min | **4 – 7.6** |
| words/min | 34 – 52 |
| 首对白延迟 | **27 – 81s**（大段世界观/奇观镜头） |
| 信息载体 | 视觉奇观 + 旁白 + 身份伏笔 |
| 爽点驱动 | 隐藏实力被低估 / 身份揭晓峰值 / 世界观震撼 |

**真实样本**：wolfless-carpenter（首对白 81s，狼人世界无狼少年）、janitor-rise-prime（首对白 64s，表彰大会造神）、kiss-vampire-lord（首对白 27s）、three-alphas（多男主玄幻，7.6）。

**写作含义**：**允许**开场 30–80s 纯画面+旁白造势（这是流派期待，不是"失败的开场"——viral-formula 的"What Fails"清单对此流派不适用）；爽点集中在身份揭晓的单一峰值，而非高频对白。对白稀疏但**每句都是身份伏笔**。

> ⚠️ **流派判定规则**：先定 Production Type（A 旁白 / B 对白，见 `production-types.md`），再用首对白延迟校准档位。**Type B + 现代背景 → 档 A/B；Type A 或玄幻/神话背景 → 档 C。** 切勿用档 A 的密度要求套档 C，反之亦然。

### §1b. 集时长预算（v3.0 修正）

v2.0 与 viral-formula 假设"每集 60–90s"，netshort 全弧实测推翻：

| 集类型 | 时长 | 字数预算（按 ~120 w/m） |
|--------|------|--------------------------|
| 高密度省钱集（宫斗/悬疑） | 57–95s | 110–190 词 |
| **常规集（默认）** | **90–150s** | **180–300 词** |
| 首集 / 揭晓峰 / 付费墙集 | 160–215s | 320–430 词 |

- 真实样本（3 剧 / 29 集全弧实测）：bankrupt 8 集 93–209s（均值 136s）；married-a-ceo 12 集 91–227s（均值 161s，付费墙 ep10=227s/ep12=210s）；wolfless 9 集 102–214s（均值 166s，付费墙 ep9=208s）。**三剧综合均值 156s，付费墙集几乎都是该弧 top-3 长集。**
- **禁止把剧本硬压进 60s**——会迫使对白密度虚高到 ~180 w/m（不自然）或砍掉情绪铺垫。
- 详见 `netshort-arc-mechanics.md` §1。

## §2. 单集爽点时序钉（钩子 → 压/放弹簧 → 再钩子）

真实样本几乎**全部**遵循同一时序骨架，与集时长（57–264s）无关：

```
0 – 6s    │ 首钩子（HOOK）—— 核心前提 / 打脸开场 / 身份伏笔，一击抓住
6 – 30s   │ 铺垫升级（SETUP）—— 抛出本集核心冲突与人物关系
30 – 45s  │ 中段反转（MID-REVERSAL）——【硬钉】新信息改变场景方向（压或放）
45 – 末5s │ 推进至峰值（ESCALATION）
末 5 –10s │ 结尾再钩子（END-HOOK）—— 比 6s 钩子更强的未竟爽点
```

**30–45s 中段反转是硬钉**（viral-formula 只说"mid-episode reversal"未给时间）。真实例证：

- **married-a-ceo ep1**（162s）：2.8s 婚姻宣布 → 21s 背叛揭露（压）→ **76s**"我宁愿路边拉个人嫁"→真拉了（放，反转）→ 158s "Wait. What?"（再钩子）
- **adopted-baby ep1**（137s）：0s 喜剧打脸开场（金山）→ **54s** 下凡被凡人骂"beggar"（压）→ **77s** "You just said no to a god"（放，反差爽点）→ **118s** "Can one thing just go right?"（再钩子）
- **janitor-prime ep1**（213s）：0–64s 纯画面造神 → **130s** "The Prime is here!"（放，身份揭晓峰值）→ 192s "you must hide your identity"（压，为后续蓄力）→ 210s "Mom."（再钩子）

### 情绪弹簧机制（压 vs 放，无中间态）

每一集要么**压弹簧**（积累负面情绪：虐、受辱、受挫、冤枉、低估），要么**放弹簧**（释放：爽、甜、解气、震撼、真相）。**禁止平推集**（既不压也不放 = 信息白开水 = 用户弃剧）。

| 弹簧方向 | 触发情绪 | 结尾再钩子形态 |
|----------|----------|----------------|
| **压** | 虐 / 气愤 / 焦急 / 委屈 | 受害方"灵魂拷问"或更深的困境 |
| **放** | 爽 / 解气 / 甜 / 震撼 | 反派惊愕定格 / 未被识破的身份悬念 |

**跨集呼吸**：连续 2 集压后，第 3 集必须放（否则用户疲劳弃剧）；付费墙集前 1–2 集建议压到极致（见 §4）。

## §3. 爽点类型库（7 类，全部带真实剧本例证）

viral-formula 覆盖了 hook/cliffhanger/karma 的*结构*，但缺**爽点的情绪引擎分类**。从 16 剧反推出 7 类核心引擎，每类标注适用流派与真实出处（时间戳）：

> **v4.0 量化重排（44 集 mlx-turbo 实证，见 `research-evidence-v4.md` §3）**——引擎优先级已用命中数硬量化：**①隐藏实力被低估 28（43%）** ≫ ②虐心-悔恨 12（18%）≈ ③打脸 11（17%）＞ ④身份揭晓 6（9%）＞ ⑤霸总护妻 2 ＞ ⑥契约婚反转 2 ＞ ⑦上帝视角反差 1。**"隐藏实力"是跨语料第一大引擎**，扮猪吃虎/隐藏身份/逆袭型必须把它做成贯穿全弧的主驱动（非点缀）。引擎还**按流派分化**：复仇打脸型 = 打脸+隐藏实力+身份揭晓三引擎堆叠（密度最高）；情感悬疑/家庭谎言型（shorttv mommy）= 虐心主导、打脸≈0。
>
> **v4.1 补充（85 部 AI仿真人剧发行元数据，见 `research-evidence-baidu.md`）**——付费墙落点新增**市场事实级硬默认**：长弧（均值 71 集）AI仿真人剧**付费墙聚类在卡点 10**（众数 44%、均值 10.4、卡点 9–12 占 75%）。netshort/shorttv 短弧（6–12 集）的付费墙工艺不变，但长弧要把"梯次揭晓"扩成**多轮**：每 ~10 集一小揭晓峰 + 虐心谷，墙前再放大。女频基底（霸总/护妻 61% + 虐心/背叛 57%）默认必含其一。
>
> **v4.2 视频级反推（netshort 44 + shorttv 87 = 131 集全量 ASR，见 `research-evidence-video.md`）**——五条**视频证据级硬默认**（全语料坐实）：①**冷开场 in-media-res**——首句对白/旁白落 **0–1s**（均值 0.26s），必须是进行时冲突，禁回忆杀/风景建立/旁白自我介绍；ep1 ≤10s 点核心前提，25s 内建观众>主角的上帝反差信息差。②**密度量化**——**≥15 segs/min、≥100 wpm**（<90 wpm 判密度不足）；hook 集（ep1–3）节拍间隔 **≤30s**。netshort 长集（~2.5min）走 ~14/100，shorttv 短集（~1.3min）走 ~21/200，靠 segs/min 维持密度。③**爽点"簇射"曲线**——免费弧 = ep1–3 高位 hook(≥3/集) → 中段复利升温(≥2/集 且不得为 0) → **最后 1–2 集付费墙簇射(≥5，多类型同场对撞)**；15 条弧无一例外在付费墙边缘集骤升到 5–10。④**付费墙悬崖双 flavor**——默认**决策/宣告悬崖**（离婚/签约/揭身份/复仇启动，payoff 落墙后），纯肉体危机仅 27% 非主流（补全 v3.0「生死危难」为关系流派的双 flavor）。⑤**中段禁零**——任意连续 2 集爽点之和 <3 即判失败模式（零爽点中段=观众流失点）。平台工艺分野：netshort=对白对峙/当场兑现/对白≥85%；shorttv=旁白内省（允许 30–40%）但不得替代爽点兑现。

### 1. 打脸反杀型（Face-Slap / Karma）
配角/反派居高临下或施虐，为后续主角逆袭的"解气"蓄力；或在集内即时打脸。
- bankrupt ep1@8s：「I'll toss you into the Pacific and feed you to the sharks」（威胁型开场，立女霸总人设）
- custody ep1@191s：「You can't even feed yourself. And you think you can fight me for Bob's custody?」（前夫夺抚养权 → 为女主反杀悔恨蓄力）
- **适用**：复仇打脸、逆袭、职场、宫斗

### 2. 身份揭晓型（Identity Reveal）
隐藏的真实身份（CEO / 霸总 / 神话身份 / 血脉）在关键节点曝光。
- janitor-billionaire ep1@150s：「the president of the zenith group, the king zenith that everyone looks up to」（窗清洁工 = 集团之王）
- janitor-prime ep1@130s：「The Prime is here!」（退隐超级士兵现身）
- custody ep1@51s：「I'm carrying the child of Rex Sinclair? The billionaire.」
- **适用**：隐藏身份霸总、逆袭、萌宝、玄幻

### 3. 霸总护妻/护崽型（Fierce Protection）
强大男主/萌宝以压倒性力量保护女主，制造"被偏爱的安全感"爽点。
- married-a-ceo ep1@113s：「I'd rather grab a stranger off the street and marry him」→ 街边即拉即结（契约婚护妻）
- alpha-daddy ep1@148s：「the other alphas are hunting you, we need to get you out of here」（Alpha King 寻妻护崽）
- beautiful-ceo ep1@61s：萌宝武力护母追逐戏
- **适用**：契约婚、霸总、狼人、萌宝

### 4. 隐藏实力被低估型（Hidden Power Underestimation）★核心引擎
主角主动隐藏或被误判为"弱者"，配角轻视 → 观众上帝视角期待揭晓时刻。**这是跨度最大的爽点引擎**（玄幻+逆袭+萌宝共用）。
- janitor-prime ep1@192s：「you must hide your identity. You are never to reveal who you are」（母亲令其隐退 → 全剧打脸蓄力总开关）
- wolfless-carpenter ep1@93s：「a wolf-less boy surpassed me in two decades」（狼人世界里无狼少年 = 最强）
- ugly-girl-cooking ep1@97s：「I'm Isabella Vance, the grand ancient witch, 10,000 years of craft」（被当死人，实为万年女巫）
- adopted-baby ep1@74s：财神萌宝被骂「dirty little beggar」
- **适用**：玄幻、狼人、吸血鬼、逆袭、隐藏身份霸总、萌宝

### 5. 虐心-悔恨型（Agony → Regret）★高频付费驱动
主角被冤枉/伤害到极致（甚至死亡），观众期待施虐者发现真相后的悔恨。**这是 mommy 类情感悬疑剧的付费转化主引擎。**
- mommy ep6@46s：Lily 被活活冻害死：「I'm really dead now. I wasn't lying. Can you forgive me now?」
- mommy ep6@82s：「If Mom finds out I'm dead, will she be sad? Will she regret it?」（灵魂拷问画外音 = 终极钩子）
- hidden-child ep1@50s：「I was just pitying you... I'm bored of playing sugar mama with a nerd」（渣女甩男主 → 为逆袭悔恨蓄力）
- **适用**：情感悬疑、萌宝认父、复仇、受虐逆袭

### 6. 契约婚反转型（Contract Marriage Reversal）
婚礼/关系节点被突发状况打破并瞬间重组。
- married-a-ceo ep1：婚礼当天被甩 → **122s 街边拉陌生霸总结婚** → 158s「Wait. What?」
- custody ep1：一夜情 → 怀孕 = 亿万富翁的孩子
- **适用**：契约婚、抚养权、霸总

### 7. 上帝视角反差型（Dramatic Irony）
观众掌握关键信息（真身/血脉/关系），剧中人不知 → 每一次误判都是爽点蓄力。
- adopted-baby：观众知萌宝是神，凡人配角每次轻视都强化"待打脸"期待
- three-alphas ep1@10s：「You possibly have three mates.」（观众知女主有 3 个 Alpha，校园不知）
- alpha-daddy ep1@173s：Alpha King 不知 Lillian 怀的是 heir
- **适用**：萌宝、玄幻、多男主、隐藏身份

> **每集至少落地 1 类引擎**，付费墙集落地 2–3 类叠加（见 §4）。

## §4. 付费卡点集密度公式（付费转化硬设计）

> **v3.0 重要分流**：付费墙有**两种模型**，按平台/流派择优。
> - **shorttv 模型**（本节，适用：重生宫斗/情感悬疑）：付费墙 = 真相即将曝光 + 蒙鼓 + 灵魂拷问。观众付费为"看反应"。
> - **netshort 模型**（见 `netshort-arc-mechanics.md` §2，适用：复仇打脸/隐藏身份/契约婚）：付费墙 = 主角最大生死危难，真相与清算全部押后。观众付费为"看救赎+清算"。
> - 写复仇/隐藏身份/契约婚 → 用 netshort 模型；写宫斗/悬疑 → 用本节 shorttv 模型。

真实样本证明：**付费墙前的卡点集，密度显著回升至中段集的 ~1.5–2×，并叠加虐点峰值 + 灵魂拷问双层 cliffhanger。**

### mommy 弧线密度序列（情感悬疑剧，6 集免费，ep7+ 付费）

| 集 | segs/min | 集位功能 | 爽点设计 |
|----|----------|----------|----------|
| ep1 | 13.7 | 高开钩子 | truth band 高概念 + 首次冤枉（引擎建立） |
| ep2 | 15.7 | 冲突升级 | — |
| ep3 | 7.9 | **密度谷** | 持续虐 + 家庭撕裂（省钱集） |
| ep4 | 8.3 | **密度谷** | 过敏迫害（省钱集） |
| ep5 | 9.0 | 蓄力 | 矛盾深化 |
| ep6 | **13.4** | **付费墙峰** | 虐点峰值（死亡）+ 真相悬决 + 灵魂拷问 |

**K 线节奏**：高开 → 中段密度谷（情绪维持、制作省钱）→ 付费墙密度峰（诱饵集，爽点加密）。

### 付费卡点集三件套（每张卡的末集必须满足）

1. **虐点/冲突峰值**：把主角推到该弧最痛的点（死亡/破产/夺子/真相被埋）。mommy ep6 = 主角死亡。
2. **真相方仍蒙在鼓里**：施虐者/误解者继续冤枉（"It's an act, seriously who's allergic to broccoli?"），上帝视角虐心拉满。
3. **双层 cliffhanger**：
   - 层一（动作）：真相方即将发现（"let her out"——妈妈即将打开门发现尸体）
   - 层二（灵魂）：受害方画外音拷问（"will she be sad? will she regret it?"）

### 卡点集密度回升规则

```
卡点集 segs/min ≥ 该剧中段集均值的 1.5×
卡点集必须叠加 ≥2 类爽点引擎（§3）
卡点集 cliffhanger 强度 > 该弧任何单集
```

> viral-formula 的 Card System 给了"Ep10/20/30 must be DEVASTATING"的*定性*要求；本文补的是**密度回升倍数**与**三件套结构**的*量化*要求。

## §5. 信息密度自检表（写完每集后强制过）

> **这是硬门禁**，不是建议。任一 ✗ → 该集重写。阈值按 §1 档位取。

### 每集自检（对白密集型 / 档 A）

- [ ] segs/min ≥ 8（复仇/宫斗/契约婚/情感悬疑）
- [ ] words/min ≥ 75
- [ ] 首对白 ≤ 10s
- [ ] 中段反转落在 30–45s 区间（或集长的 25%–45% 处）
- [ ] 至少落地 1 类 §3 爽点引擎
