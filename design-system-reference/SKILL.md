## Design Principles

### 1. Clarity Over Cleverness
- Make the obvious, obvious
- Use conventional patterns
- Test with real users

### 2. Accessibility First
- WCAG AA minimum contrast
- Keyboard navigation
- Screen reader support
- Focus indicators

### 3. Performance Matters
- Optimize images and assets
- Lazy load content
- Minimize JavaScript
- Test on slow connections

### 4. Mobile-First
- Design for smallest screen first
- Progressive enhancement
- Touch-friendly targets (44x44px minimum)
- Responsive breakpoints

### 5. Consistent Systems
- Design tokens for all values
- Component libraries
- Reusable patterns
- Documented guidelines

## Common Anti-Patterns

❌ **Don't do this:**
- Low contrast text on backgrounds
- Generic purple gradient on white
- Overused fonts (Inter, Roboto everywhere)
- Inconsistent spacing
- No hover states
- Missing loading states
- Tiny touch targets
- Hamburger menus for main navigation

✅ **Do this instead:**
- High contrast ratios (4.5:1 minimum)
- Unique color schemes per project
- Context-appropriate typography
- Systematic spacing scales
- Interactive feedback on all elements
- Skeleton screens and loaders
- Large touch targets (44x44px+)
- Visible navigation when possible
