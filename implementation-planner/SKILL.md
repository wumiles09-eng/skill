# Implementation Plan Writer - 实施计划编写

## 目标

将设计方案转化为可执行的实施计划，确保开发有序进行。

## 实施计划结构

```markdown
# {功能名称} 实施计划

## 1. 概述

### 1.1 目标
{要达成的目标}

### 1.2 范围
- 包含：{功能范围}
- 不包含：{明确排除}

### 1.3 约束
- 时间：{截止日期}
- 资源：{人力/预算}
- 技术：{技术栈限制}

## 2. 任务拆解

### 2.1 任务列表
| 任务ID | 任务名称 | 预估时间 | 负责人 | 依赖 | 状态 |
|--------|---------|---------|--------|------|------|
| T-1 | {任务1} | {时间} | {人员} | - | 待开始 |
| T-2 | {任务2} | {时间} | {人员} | T-1 | 待开始 |
| T-3 | {任务3} | {时间} | {人员} | T-2 | 待开始 |

### 2.2 任务详情
#### T-1: {任务名称}
**描述**: {任务描述}
**输入**: {需要的输入}
**输出**: {预期产出}
**验证**: {如何验证完成}
**风险**: {潜在风险}

## 3. 测试计划

### 3.1 测试范围
- 单元测试：{测试点}
- 集成测试：{测试点}
- E2E测试：{测试点}

### 3.2 测试用例
| 用例ID | 场景 | 步骤 | 预期结果 |
|--------|------|------|---------|
| TC-1 | {场景} | {步骤} | {结果} |

## 4. 风险与预案

| 风险 | 概率 | 影响 | 缓解措施 | 预案 |
|------|------|------|---------|------|
| {风险1} | 高/中/低 | 高/中/低 | {措施} | {预案} |

## 5. 里程碑

| 里程碑 | 日期 | 交付物 | 验收标准 |
|--------|------|--------|---------|
| M1 | {日期} | {产出} | {标准} |
| M2 | {日期} | {产出} | {标准} |

## 6. 回滚计划

### 6.1 回滚触发条件
- [ ] {条件1}
- [ ] {条件2}

### 6.2 回滚步骤
1. {步骤1}
2. {步骤2}

### 6.3 回滚验证
- [ ] {验证项1}
- [ ] {验证项2}
```

## 任务拆解原则

### 1. 任务粒度

```markdown
## 任务粒度标准

### 合适的任务（5-20分钟）
- ✅ "实现登录表单UI"
- ✅ "添加登录API端点"
- ✅ "编写登录单元测试"

### 太大的任务（需要拆分）
- ❌ "实现用户认证系统" → 拆分为多个小任务
- ❌ "完成订单功能" → 拆分为订单创建/支付/状态等

### 太小的任务（需要合并）
- ❌ "创建文件" → 合并到相关任务中
- ❌ "写一个函数" → 合并到功能任务中
```

### 2. 任务依赖

```markdown
## 依赖关系

### 串行依赖
```
T-1 → T-2 → T-3
```

### 并行任务
```
    T-2a
T-1 →
    T-2b
```

### 复杂依赖
```
T-1 → T-2a → T-3
     ↘ T-2b ↗
```
```

## 验证方式

### 代码验证

```markdown
## 代码验证清单

### 自动化验证
- [ ] 代码格式化通过
- [ ] Lint检查通过
- [ ] 类型检查通过
- [ ] 单元测试通过

### 手动验证
- [ ] 代码审查完成
- [ ] 逻辑正确性确认
- [ ] 边界条件检查
```

### 功能验证

```markdown
## 功能验证清单

### 基本功能
- [ ] 正常路径可用
- [ ] 输入验证正常
- [ ] 错误处理正常

### 边界情况
- [ ] 最小值处理
- [ ] 最大值处理
- [ ] 空值处理
- [ ] 异常值处理
```

## 风险评估

### 风险矩阵

```
影响
  高 │   │   │ ✓ │ ✓ │
  中 │   │ ✓ │ ✓ │ ✓ │
  低 │   │ ✓ │ ✓ │   |
     └───┴───┴───┴───┴─→ 概率
        低  中  高
```

### 风险应对

```markdown
## 风险应对策略

### 高概率+高影响（必须处理）
- **风险**: {描述}
- **缓解**: {具体措施}
- **预案**: {备选方案}

### 低概率+高影响（准备预案）
- **风险**: {描述}
- **监控**: {监控指标}
- **预案**: {备选方案}

### 高概率+低影响（接受或简化）
- **风险**: {描述}
- **策略**: {接受/简化}
```

## 里程碑管理

```markdown
## 里程碑设置

### 原则
- 每个里程碑有明确交付物
- 里程碑间隔1-2周
- 有清晰的验收标准

### 示例里程碑

#### M1: 需求确认（Week 1）
**交付物**: PRD文档
**验收**: PM确认签字

#### M2: 设计完成（Week 2）
**交付物**: 设计文档 + 原型
**验收**: 团队评审通过

#### M3: 开发完成（Week 4）
**交付物**: 可运行的代码
**验收**: 所有测试通过

#### M4: 上线发布（Week 5）
**交付物**: 生产环境运行
**验收**: 无P0/P1 bug
```
